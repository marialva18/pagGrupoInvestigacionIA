# Sprint 0

## Objetivo

Dejar una base común, ejecutable, segura y documentada antes de repartir módulos.

## Primer flujo técnico

1. Instalar dependencias.
2. Ejecutar web, API y worker.
3. Abrir la portada.
4. Consultar `/api/v1/health`.
5. Diseñar el esquema de datos.
6. Implementar autenticación y permisos.

## No se incluye aún

- Modelo Prisma definitivo.
- Autenticación real.
- Editor completo.
- Integración efectiva con Mailjet.
- Integración efectiva con Supabase.
- Ingesta de noticias.
